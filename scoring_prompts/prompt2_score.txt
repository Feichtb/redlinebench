# RedlineBench — Scoring Prompt

You are scoring AI model responses to the RedlineBench architectural drawing review benchmark. Score each model from scratch by reading its output directly against the benchmark answer key — do not rely on any triage files.

## Scoring Rubric

For each scoreable issue in the answer key (categories A through G):

- **1.0** — The model found this issue and described it correctly and specifically. The key information is present: what is wrong, where it appears (sheet/location), and why it matters (or what it conflicts with).
- **0.5** — The model found this issue but was vague, incomplete, or only partially correct. For example: flagged the right area but wrong reason, or described the symptom without identifying the conflict.
- **0.0** — The model did not flag this issue, or what it said is clearly unrelated.

**Incorrect Findings penalty: -1.0 each**
If the model makes a finding that appears on the Known Incorrect Findings list AND presents it with confidence (not tentatively or hedged), deduct 1.0. If the model hedges ("this may be..." or "I'm not certain..."), do not penalize.

**Neutral Findings: no score impact**
If the model flags something on the Neutral list, count it in the neutral tally but do not add or subtract points.

---

## Instructions

### Step 1 — Determine max_possible

Count the total number of scoreable issues in the answer key by summing all issues in categories A through G. Do not use a hardcoded number — the answer key grows over time. Record this as `max_possible`.

### Step 2 — Score Each Model

For each model output file, read through the entire response. Identify each discrete finding. Then work through every issue in the answer key in order and score the model against it.

For each issue:
1. Check whether the model output contains a finding that matches this issue.
2. Assign a score: 1.0, 0.5, or 0.0 based on the rubric above.
3. Note a brief justification when the score is anything other than 0.

If the model mentions the same issue in multiple places, score based on the best/clearest instance.

The model output may use its own numbering (e.g., "A-1", "B-3") — this is the model's own labeling, NOT the answer key IDs. Score by content, not by label.

### Step 3 — Tally Incorrect Findings

For each model, check all findings against the Known Incorrect Findings list. Apply -1.0 penalty if stated with confidence.

### Step 4 — Tally Neutral Findings

Count findings matching the Neutral list. No score impact.

### Step 5 — Count Total Items Flagged

Count the total number of distinct findings the model reported, regardless of how they score.

### Step 6 — Calculate Scores

- **Category scores**: Sum of issue scores within each category (A through G)
- **Raw score**: Sum of all issue scores
- **Net score**: Raw score minus total incorrect finding penalty
- **Percent**: (Net score / max_possible) × 100, rounded to one decimal

## Important Reminders

- Score based on the finalized answer key.
- The model output may have its own numbering (e.g., "A-1" through "G-10") — these are the model's own labels, not answer key IDs. Score by content, not label.
- Score conservatively: give 1.0 only if the specific conflict is clearly identified. Give 0.5 if the model found the right area but was missing key details. Give 0 if unclear.
- Do not award 0.5 out of sympathy. The model either added useful, specific information or it didn't.
- When in doubt about whether a finding matches an answer key issue, go back to the answer key criteria.
